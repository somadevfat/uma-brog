# 技術スタック選定書: 堅牢なフルスタックポートフォリオブログ

## 1. コンセプト

「高速」「堅牢」「拡張性」を最優先事項とし、エッジコンピューティングを最大限に活用したモダンなフルスタックブログを構築します。

## 2. コアスタック

| カテゴリ           | 選定技術          | 特徴・採用理由                                                                                               |
| :----------------- | :---------------- | :----------------------------------------------------------------------------------------------------------- |
| **ランタイム**     | **Bun**           | Node.jsより高速な実行環境、標準搭載のテストランナー・パッケージマネージャーを活用。                          |
| **フレームワーク** | **HonoX**         | Honoの高速なルーティングとViteの柔軟性を統合。Islandsアーキテクチャによる最小限のJS配信。                    |
| **View Engine**    | **Hono JSX**      | Reactライクな記述が可能。依存が少なく、サーバーサイドレンダリング(SSR)において圧倒的なパフォーマンスを発揮。 |
| **ORM**            | **Drizzle ORM**   | TypeScriptファースト、SQLに近い直感的な記述、極めて薄いオーバーヘッド。                                      |
| **データベース**   | **Cloudflare D1** | Cloudflareのエッジ上で動作するマネージドSQLite。Honoとの相性が最高。                                         |

## 3. 開発・品質管理

| カテゴリ           | 選定技術            | 特徴・採用理由                                                                                            |
| :----------------- | :------------------ | :-------------------------------------------------------------------------------------------------------- |
| **バリデーション** | **Zod**             | `@hono/zod-validator` を使用し、リクエストからDB層まで一貫した型安全性を確保。                            |
| **コンテンツ管理** | **MDX (Gitベース)** | 記事をMarkdownで管理しつつ、JSXコンポーネントを埋め込める拡張性を確保。                                   |
| **スタイリング**   | **Vanilla CSS**     | 外部ライブラリに依存せず、CSS Variablesと `@layer` を用いて、軽量かつ堅牢なデザインシステムを自前で構築。 |
| **認証**           | **Better Auth**     | Bun/Hono環境に最適化された最新の認証ライブラリ。                                                          |
| **テスト**         | **Vitest**          | Viteベースのビルド環境と統合された高速なテスト環境。                                                      |

## 4. アーキテクチャの方針

### 4.1. Islands Architecture

- 基本的にすべてのページはSSRによる静的なHTMLとして配信。
- インタラクティブな要素（検索ダイアログ、動的な演出など）のみを `islands/` ディレクトリ配下で定義し、クライアントサイドでハイドレート。

### 4.2. 型安全性の貫徹 (End-to-End Type Safety)

- Drizzleのスキーマからバリデーション用のZodスキーマを生成。
- RPC機能（もし必要なら）や、`HonoX` の型定義を活用し、サーバーからフロントまで型定義の乖離を防ぐ。

### 4.3. パフォーマンス最適化

- バンドルサイズの最小化。
- Cloudflare Pages によるエッジ配信とR2（画像保存用）の活用を視野に入れる。

## 5. デプロイ構成

- **Hosting**: Cloudflare Pages
- **Database**: Cloudflare D1
- **Storage**: Cloudflare R2 (オプション: 必要に応じて画像保存に使用)
- **CI/CD**: GitHub Actions

---

## 決定事項

本プロジェクトは、上記技術スタックに基づき **Bun** を標準ランタイムとして開発を進行する。
フロントエンドは **Hono JSX** を採用し、ピュアなHTMLのパフォーマンスとJSXの開発体験を両立させる。
