# コーディングルール共通

- DRY SOLIDを意識
- JSDOCを日本語で必ずすべて記載 {型} 説明と処理の説明
- 実装ひとまとまりをコメントを1行でコメントで説明する。
- any禁止
- **フロント・バックエンド間の型共有 (Full-stack Type Safety)**: 
  - API 通信には Hono RPC (`hc`) を必ず使用し、URL 文字列の直接記述を避ける。
  - バリデーション用 Zod スキーマを双方で共有し、DRY 且つ安全な実装を行う。
- テストを通すために無理やり挙動の変わる修正をしない。
- 型は既存のものを優先的に使う
- アンチパターンは使わない。
