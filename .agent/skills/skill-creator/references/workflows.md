# ワークフローパターン (Workflow Patterns)

## シーケンシャルワークフロー (Sequential Workflows)

複雑なタスクの場合は、操作を明確な順次ステップに分割してください。`SKILL.md` の冒頭近くでプロセスの概要をClaudeに伝えると非常に役立ちます。

```markdown
PDFフォームの入力には以下のステップが含まれます：

1. フォームを分析する (`analyze_form.py` を実行)
2. フィールドマッピングを作成する (`fields.json` を編集)
3. マッピングを検証する (`validate_fields.py` を実行)
4. フォームに入力する (`fill_form.py` を実行)
5. 出力を確認する (`verify_output.py` を実行)
```

## 条件付きワークフロー (Conditional Workflows)

分岐ロジックを伴うタスクの場合は、Claudeを決定ポイントに導いてください：

```markdown
1. 修正のタイプを決定します：
   **新しいコンテンツを作成しますか？** → 以下の「作成ワークフロー」に従ってください
   **既存のコンテンツを編集しますか？** → 以下の「編集ワークフロー」に従ってください

2. 作成ワークフロー: [ステップ]
3. 編集ワークフロー: [ステップ]
```
