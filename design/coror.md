# デザイン要件定義書：テクニカル・ブループリント・スタイル

## 1. コンセプト

- **コアテーマ:** レトロフューチャー、宇宙工学、極限のミニマリズム、冷徹な精密さ
- **視覚的特徴:** 暗闇に浮かび上がる細い線画、計器類のようなUI、無機質な余白の美
- **ユーザーに与える印象:** 専門的、機密情報、スタイリッシュ、緊張感

## 2. カラーパレット

全体の95%をモノトーン（黒・白・グレー）で構成し、残り5%のアクセントカラーで視線を強烈に誘導します。

| 役割                    | カラーコード                | 用途・備考                                                                 |
| ----------------------- | --------------------------- | -------------------------------------------------------------------------- |
| **Main Background**     | `#0A0A0A`                   | 漆黒に限りなく近いダークグレー。完全な黒（`#000000`）より線の白が馴染む。  |
| **Primary Line/Text**   | `#FFFFFF`                   | 主要なテキスト、見出し、前面の図解の線。100%の白でコントラストを出す。     |
| **Secondary Line/Text** | `#666666`                   | サブテキスト、枠線、非アクティブな要素。                                   |
| **Grid Background**     | `rgba(255, 255, 255, 0.05)` | 背景の方眼グリッド線。極めて薄く設定し、ノイズにならないようにする。       |
| **Accent (Red)**        | `#E60012`                   | 警告、強調、重要なリンク、ボタンのホバー状態。彩度の高い「血のような赤」。 |
| **Accent (Tan)**        | `#D4C4A8`                   | （オプション）図解内の明かりや、少し有機的な要素を足したい時のベージュ。   |

## 3. タイポグラフィ

「文字も図形の一部」として扱います。太いフォントは避け、細く幾何学的なものを選択します。

- **欧文フォント (メイン):** `Roboto Mono`, `Share Tech Mono`, または `Inter`
- **ウェイト:** `Thin` (100) または `Light` (300)
- **特徴:** 等幅（Monospace）フォントを使うことで、ターミナルや計器のような印象を強調。
- **字間 (Letter Spacing):** `0.1em` 〜 `0.2em`。字間を広くとることで、冷たい緊張感を演出。

- **和文フォント (サブ):** `Yu Gothic (游ゴシック)` または `Noto Sans JP`
- **ウェイト:** `Light` または `Thin`
- **特徴:** 丸みのない、シュッとしたモダンなゴシック体を推奨。あえて明朝体（`Noto Serif JP`など）を使うと、ネオ・クラシカルで少し不気味な（エヴァンゲリオンのような）雰囲気が増します。

## 4. UIコンポーネント・装飾ルール

- **背景グリッド (方眼紙効果):**
- 常に背景に20px〜40px角の極細グリッド（`1px`、不透明度`5%`）を敷き詰める。空間の広がりと設計図らしさを出す要。

- **ボーダー（罫線）:**
- 要素を区切る線、カードの枠線はすべて `1px` の極細線（Solid）を使用。
- 角丸（`border-radius`）は基本使用しない（`0px`）、または極端に小さくする（`2px`程度）ことで、ソリッドな工業製品感を出す。

- **ボタン / リンク:**
- 塗りつぶしのボタンは極力避ける。アウトライン（枠線のみ）のボタンを基本とする。
- ホバーアクションで、枠線とテキストが `Accent (Red)` に変化し、わずかに光る（`box-shadow`）エフェクトを入れる。

- **画像・イラスト:**
- 写真はそのまま使わず、モノクロ加工やハーフトーン（網点）加工を施す。
- 理想はSVGによるワイヤーフレーム（線画）イラスト。

## 5. レイアウト構造

- **余白 (Margin/Padding):**
- コンテンツ間の余白は均等かつ広めにとる。「無」の空間をデザインとして活かす。

- **配置:**
- 情報を中央揃えにするか、グリッドに沿って厳密に左・右へ整列させる。ランダムな配置は避け、数学的な規則性を持たせる。
